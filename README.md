# ImageMove

ImageMove は、画像を目視確認しながら除外先フォルダへ安全に振り分ける Windows 向け WinForms ツールです。既存の `1枚ずつ確認` モードに加えて、まとめて選別しやすい `グリッド目検` モードを追加しました。

重要:

- 元画像を物理削除しません。
- 除外は既存運用に合わせて除外フォルダへの移動で行います。
- `Ctrl+Z` または `元に戻す` で直前操作をアンドゥできます。
- 日本語パス、日本語ファイル名、スペース入りパスを前提に扱えます。

## 起動方法

- 既存のリリース実行ファイル: `release\ImageMove.exe`
- ビルド: `build_release.bat`

`release\ImageMove.exe` を起動すると、同じフォルダ配下に `setting.xml` が保存されます。

## モード切替

画面上部の `確認モード` で切り替えます。

- `1枚ずつ確認`
  - 従来どおり 1 枚ずつ確認して移動します。
- `グリッド目検`
  - サムネイルを一覧表示し、複数枚をまとめて除外または戻しできます。

既存モードとグリッド目検モードは同じ移動履歴を共有します。グリッドで除外した画像は既存モード側でも除外済みとして扱われ、グリッドで戻した画像は既存モード側でも復帰状態になります。

## グリッド目検モードの使い方

1. 元フォルダを開き、画像を読み込みます。
2. `確認モード` を `グリッド目検` に切り替えます。
3. 右側の表示設定でサムネイルサイズ、1ページ表示件数、表示項目を調整します。
4. 各サムネイルはチェックボックスだけでなく、画像本体をクリックしてもチェックを切り替えられます。ダブルクリックすると `1枚ずつ確認` 側で開きます。
5. 右上の `実行` 欄にある `チェック画像を除外へ送る` を押し、移動先フォルダを確認して実行します。
6. 必要なら `元に戻す` または `チェック画像を戻す` で戻します。

各サムネイルには次の情報を表示できます。

- チェックボックス
- サムネイル画像
- 短縮ファイル名
- 画像 ID

画像サイズ、ファイルサイズ、現在パスはツールチップで確認できます。

画面右上には常に次の件数を表示します。

- `選択中`
- `表示中`
- `全体`

## 複数選択

最低限必要な複数選択操作を実装しています。

- 個別チェック
- 全選択
- 全解除
- 表示中のみ全選択
- 選択反転
- チェック済みのみ表示
- 未チェックのみ表示

チェック状態はページ移動、並び替え、絞り込みをまたいでも保持されます。保持キーは元画像の元パスです。

## 1ページ最大 1000 件

`1ページ表示件数` は次を選べます。

- 100
- 300
- 500
- 1000
- カスタム

初期値は `1000` です。

1000 件表示でも固まりにくくするため、次の方式を使っています。

- 現在ページ分だけを `ListView` に構築
- サムネイルはバックグラウンド生成
- 進捗表示あり
- サムネイル生成のキャンセルあり
- メモリキャッシュ + ディスクキャッシュ
- 画像サイズ系の絞り込み時だけ必要なメタデータを収集
- 壊れた画像は失敗扱いで継続

注意:

- 1000 件表示時は、表示直後にサムネイル生成が走るため、初回だけ待ち時間が発生します。
- `画像サイズ順`、`縦横比`、`画像サイズ` 絞り込みは、必要に応じてサイズ情報の収集を行います。

## サムネイル設定

表示設定で次を変更できます。

- サムネイルサイズ
  - 小
  - 中
  - 大
  - カスタム
- 1ページ表示件数
  - 100
  - 300
  - 500
  - 1000
  - カスタム
- ファイル名を表示
- 画像 ID を表示
- チェックボックスを表示

初期値:

- サムネイルサイズ: 中
- 1ページ表示件数: 1000
- ファイル名表示: ON
- 画像 ID 表示: ON
- チェックボックス表示: ON

## 並び替えと絞り込み

並び替え:

- 既存順
- ファイル名順
- 更新日時順
- 作成日時順
- 画像サイズ順
- ファイルサイズ順

絞り込み:

- ファイル名検索
- 拡張子
- 縦横比
- 画像サイズ
- チェック済みのみ / 未チェックのみ
- 状態
  - 未処理
  - 除外済み
  - 戻し済み

類似画像順や pHash / dHash / aHash による類似グループ表示は、今回の初期実装では未対応です。

## ページング

画像数が 1000 件を超える場合はページングします。

- `前へ`
- `次へ`
- `現在ページ`
- `ページ x / y`
- 1ページ件数変更

チェック状態はページをまたいでも保持します。

## 一括除外と戻し

主な操作:

- `チェック画像を除外へ送る`
- `チェック画像を戻す`
- `選択を解除`

これらの実行ボタンは、右側パネル上部の `実行` 欄へ固定表示されます。スクロールして探す必要はありません。

一括除外時は次を確認できます。

- 対象枚数
- 除外先フォルダ
- 実行可否

安全策:

- 物理削除しません。
- 上書きしません。
- 同名ファイルがある場合は既存ロジックに従って安全側でスキップします。
- 失敗したファイルはログへ残します。
- 処理中に閉じようとした場合は確認します。

## アンドゥ

`元に戻す` ボタンまたは `Ctrl+Z` で直前操作を元に戻せます。

対象:

- 直前の一括除外
- 直前の一括戻し
- 既存モードからの移動操作

内部では操作単位で履歴を持ち、各ファイルについて次の情報を扱います。

- 操作 ID
- 操作種別
- 実行日時
- 移動元
- 移動先
- 元の並び位置
- 成功/失敗

制限:

- Undo 履歴の永続化 JSON 保存は未実装です。
- 履歴はアプリ終了まで保持します。

## サムネイルキャッシュ

サムネイルキャッシュは 2 段構成です。

- メモリキャッシュ
  - LRU 方式
  - 件数上限とメモリ上限あり
- ディスクキャッシュ
  - 保存先: `%LOCALAPPDATA%\ImageMove\thumbnail_cache`
  - 元ファイルパス、サイズ、更新時刻、サムネイルサイズを含めたキーで再利用

`キャッシュをクリア` を押すと、メモリとディスクの両方を削除して再生成します。

## ログ

グリッド目検モードのログは実行ファイルのある場所の `logs` フォルダへ出力します。

- `logs\imagemove_grid_review_YYYYMMDD_HHMMSS.log`

主な記録内容:

- 画像一覧のグリッド同期開始/完了
- サムネイル生成開始/完了/停止
- 画像サイズ情報の収集開始/完了/停止
- 一括除外
- 一括戻し
- アンドゥ
- 対象枚数
- 成功数
- 失敗数
- 移動元
- 移動先
- エラー内容

## 設定保存

既存の `setting.xml` を拡張し、次を保存します。

- 最後に開いたフォルダ
- 除外先フォルダ
- サムネイルサイズ
- 1ページ表示件数
- 並び順
- ファイル名表示 ON/OFF
- 画像 ID 表示 ON/OFF
- チェックボックス表示 ON/OFF
- ウィンドウサイズ
- グリッドモードの最後の状態
- グリッド履歴列幅
- サイドバー幅
- 各種絞り込み条件
- 現在ページ

## トラブル時の確認事項

- `グリッド目検` に切り替えても画像が出ない
  - 元フォルダに対応拡張子の画像があるか確認してください。
  - `状態` や `チェック状態` の絞り込みが残っていないか確認してください。
- 初回だけ表示が遅い
  - サムネイル生成または画像サイズ情報の収集中です。右側の進捗表示を確認してください。
- サムネイルが古い
  - `キャッシュをクリア` を実行してください。
- 戻せない画像がある
  - 戻し先に同名ファイルがないか、ログにエラーが出ていないか確認してください。

## 既知の制限

- 類似画像の自動グループ化は未実装です。
- Shift 範囲選択、Ctrl 個別選択、ドラッグ範囲選択の専用実装は未対応です。
- Undo 履歴の JSON 永続化とクラッシュ復旧は未対応です。
